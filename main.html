Cours Réseaux

Notes claires et simples sur les notions de base en réseaux.

Topologies des réseaux

Définition courte :

La topologie décrit comment les équipements sont connectés physiquement ou logiquement.

  • Bus : un seul câble partagé par tous les appareils. Avantages : simple et peu coûteux. Inconvénients : collisions possibles, panne du câble impacte tout.
  • Anneau : les appareils forment une boucle. Avantages : accès ordonné au média. Inconvénients : rupture casse la communication.
  • Étoile : chaque poste relie à un central (switch). Avantages : facile à gérer et dépanner. Inconvénients : le central est point unique de défaillance.

Équipements des réseaux

Principaux éléments et rôles :

  • Hub : répète le signal sur tous les ports (niveau physique).
  • Switch : achemine les trames vers le port correct basé sur l'adresse MAC (niveau liaison).
  • Routeur : connecte et achemine entre réseaux différents (niveau réseau, IP).
  • Serveur : machine qui fournit des services (DHCP, DNS, HTTP, fichiers...).

Modèle OSI

Principe général :

Le modèle OSI divise la communication en 7 couches pour organiser les fonctions réseau et faciliter le dépannage.

  1. 7 - Application : services et applications (ex : navigateur).
  2. 6 - Présentation : formatage des données, chiffrement.
  3. 5 - Session : gestion des sessions/connexions.
  4. 4 - Transport : TCP/UDP, fiabilité, ports.
  5. 3 - Réseau : adressage logique et routage (IP).
  6. 2 - Liaison : trames, adresses MAC, contrôle d'accès au média.
  7. 1 - Physique : câbles, signaux, connecteurs.

Classes d'adresses (NetID / HostID)

Structure IPv4 en classes A à C :

Une adresse IPv4 se compose d'une partie réseau (NetID) et d'une partie hôte (HostID) suivant la classe.

  • Classe A : plages 1.0.0.0 à 126.255.255.255 — NetID = 8 bits, HostID = 24 bits.
  • Classe B : plages 128.0.0.0 à 191.255.255.255 — NetID = 16 bits, HostID = 16 bits.
  • Classe C : plages 192.0.0.0 à 223.255.255.255 — NetID = 24 bits, HostID = 8 bits.

Masque de sous-réseau (écriture par classe)

Masques par défaut pour chaque classe :

  • Classe A : 255.0.0.0 (notation CIDR : /8) — 255.0.0.0
  • Classe B : 255.255.0.0 (notation CIDR : /16) — 255.255.0.0
  • Classe C : 255.255.255.0 (notation CIDR : /24) — 255.255.255.0

Trouver l'adresse réseau (méthode)

Méthode simple :

Fais un AND bit à bit entre l'adresse IP et le masque. En pratique, pour repères rapides, mets à zéro la partie hôte selon le masque.

Exemple : IP = 192.168.1.130 et masque = 255.255.255.0 → adresse réseau = 192.168.1.0

CIDR et divisions en sous-réseau

Notation et idée :

CIDR indique le nombre de bits du préfixe réseau après une barre oblique, par exemple 192.168.1.0/24.

Plus le nombre après la barre augmente, plus les sous-réseaux sont petits (moins d'hôtes par sous-réseau).

  • /24 → 256 adresses totales (254 hôtes utilisables).
  • /25 → divise le /24 en 2 sous-réseaux (128 adresses chacun).
  • /26 → divise le /24 en 4 sous-réseaux (64 adresses chacun).

Principales balises HTML

Rôles des balises de base :

  • <!DOCTYPE html> : indique la version HTML (HTML5).
  • <html> : racine du document.
  • <head> : métadonnées, charset, liens CSS/JS.
  • <title> : titre affiché dans l'onglet du navigateur.
  • <meta> : informations comme charset et viewport.
  • <body> : contenu visible de la page.
  • <header>, <main>, <section>, <footer> : éléments sémantiques pour structurer la page.
  • <h1>…<h6> : titres hiérarchiques.
  • <p> : paragraphe de texte.
  • <ul>, <ol>, <li> : listes non ordonnées / ordonnées.
  • <a> : lien hypertexte.
  • <img> : image (balise vide avec attribut src).
